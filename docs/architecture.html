Data Flow Architecture

Request-Response Cycle

1. User Interaction - User selects filters (warehouse, category, time period, status) in Vue 3 dashboard
2. State Update - Vue reactive refs store filter selections in component state
3. API Request - client/src/api.js constructs query parameters and sends HTTP GET request to FastAPI endpoint
4. Backend Processing - FastAPI endpoint receives request, applies filters to in-memory JSON data
5. Validation - Pydantic models validate data structure and types before response
6. Response - Filtered data returned as JSON with 200 status code
7. Frontend Rendering - Vue computed properties process response data and render updated dashboard

Filter System

The system supports 4 main filter dimensions that apply across all data endpoints:

  • Warehouse - Filter inventory and orders by location (e.g., "Boston", "Denver")
  • Category - Filter by product type (e.g., "Circuit Boards", "Sensors")
  • Time Period - Filter by month or quarter (e.g., "2025-01" or "Q1-2025")
  • Order Status - Filter by fulfillment status (e.g., "Delivered", "Processing", "Backordered")

Data Sources

All data is loaded from JSON files at server startup via server/mock_data.py:

  • inventory.json - Product SKUs, quantities, locations, costs
  • orders.json - Customer orders with status and dates
  • demand_forecasts.json - Predicted demand trends
  • backlog_items.json - Unfulfilled order items
  • spending.json - Monthly expenditure data
  • transactions.json - Individual transaction records
  • purchase_orders.json - Supplier orders

Key Capabilities

Dashboard Analytics

Real-time summary statistics with interactive filtering across warehouse, category, order status, and time period dimensions. Visual metrics for inventory levels, order volume, and spending trends.

Inventory Management

Track products across multiple warehouses with stock levels, reorder points, and location data. Filter by product category and warehouse location.

Order Tracking

Monitor order status (Delivered, Shipped, Processing, Backordered) with timeline visibility. Support for multiple items per order and expected delivery dates.

Demand Forecasting

View predicted demand trends with trend indicators (up, down, stable). Support for both current and forecasted demand by product.

Backlog Management

Track unfulfilled orders with visibility into quantity discrepancies and days delayed. Prioritize items based on availability gaps.

Spending Analytics

Monitor expenditures with monthly trends, category breakdown, and transaction-level detail. Support for purchase orders and supplier management.

Architectural Decisions

In-Memory Data

Trade-off: Fast reads and simple implementation, but data doesn't persist across restarts. Suitable for demos and prototypes. Production would require a database.

Vue 3 Composition API

Benefits: Better code organization, easier logic reuse, cleaner component structure compared to Options API. Enables modern JavaScript patterns and TypeScript support.

Serverless-Ready

Design: FastAPI is lightweight and stateless, making it easy to deploy to serverless platforms. No persistent connections or background jobs.

CORS-Enabled

Development: CORS middleware allows frontend and backend to run on different ports during development. Restricted to specific origins in production.

Type Safety

Backend: Pydantic models ensure data validation and serialization. Frontend uses Vue's reactive system for runtime checks.

Separation of Concerns

Architecture: Clear separation between frontend views, API client, backend endpoints, and data layer. Easy to modify each independently.

Performance Characteristics

Current Optimizations

  • In-Memory Filtering - O(n) filtering in Python is fast for demo datasets
  • Computed Properties - Vue caches computed results until dependencies change
  • Lazy Loading - Components load data on demand via API calls
  • Vite Dev Server - Hot module replacement for rapid development iteration

Scaling Considerations

  • Database Layer - Replace JSON + in-memory with PostgreSQL or MongoDB for large datasets
  • Indexing - Add database indexes on warehouse, category, status fields for faster queries
  • Pagination - Implement cursor-based pagination for large result sets
  • Caching - Add Redis for frequently accessed dashboard summaries
  • API Rate Limiting - Prevent abuse with request throttling
